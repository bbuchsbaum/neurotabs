## For AI Agents

### Working In This Directory
- **Never edit** `RcppExports.cpp` directly. It is regenerated by `Rcpp::compileAttributes()`.
- After any change to `*.cpp` files, run: `Rscript -e 'Rcpp::compileAttributes()'`
- This regenerates both `src/RcppExports.cpp` and `R/RcppExports.R`.
- Compiled artifacts (`.o`, `.so` files) are build outputs — do not commit them.

### Testing Requirements
- C++ changes should be tested through the R test suite since Rcpp functions are called from R:
  ```r
  Rscript -e 'devtools::test(filter = "backend-nifti")'
  ```
- Build and check with: `Rscript -e 'devtools::check()'`

### Common Patterns
- Use `[[Rcpp::export]]` attribute to expose functions to R.
- Functions exposed to R are automatically wrapped in `R/RcppExports.R`.
- Rcpp headers are included via `LinkingTo: Rcpp` in `DESCRIPTION`.
